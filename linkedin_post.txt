PROJECT LAUNCH: I built an AI analyst that answers business questions about real retail data.

Not a chatbot. Not a demo with hardcoded responses.

An actual agent that:
→ Receives a plain-English question
→ Decides which of 14 analytics tools to call
→ Queries the real data (customers, KPIs, pricing)
→ Returns a grounded, business-readable answer

Ask it "how much revenue is at risk from churning customers?"
It calls the RFM segmentation tool and comes back with:
$1.64M across 189 customers — with the breakdown and a recommended action.

Ask it which products have the biggest pricing opportunity vs. what changes if you optimise for revenue instead of margin.
It calls two separate tools, notices the objectives produce opposite recommendations (Electronics: raise to $215 for margin, drop to $112 for revenue), and explains why.

That's the part I find most interesting — it connects three separate analytics systems that previously had no way to talk to each other.

The tech:
— Anthropic Claude API (native tool use, no framework overhead)
— 14 Python functions across customer, KPI, and pricing data
— Gradio chat interface
— About 100 lines of agent logic

This is Project 4 of a 4-project retail analytics portfolio I've been building.
Projects 1–3 built the data layer (RFM segmentation, KPI anomaly detection, ML dynamic pricing).
Project 4 puts a conversational interface on top of all of them.
